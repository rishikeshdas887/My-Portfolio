Security by Design

Hardened on purpose.

This portfolio is intentionally engineered to be a small, auditable target. No accounts, no databases, no third-party trackers — just strict browser policies and a minimal, verifiable surface area. Practices I ship in every product I build.

🛡️

Strict Content-Security-Policy

Locked-down CSP restricts scripts, styles, fonts, images and connections to trusted origins. Mitigates XSS at the browser.

🚫

No Third-Party Trackers

Zero analytics scripts, zero social pixels, zero unknown SDKs. Fewer suppliers = smaller attack surface.

🔒

Hardened Response Headers

X-Content-Type-Options nosniff, Referrer-Policy strict-origin, Permissions-Policy zeroed, frame-ancestors none — clickjacking blocked.

🕵️

No Auth = No Auth Vulns

This site holds no accounts, sessions or PII. No logins to phish, no tokens to steal, no database to breach.

🌐

HTTPS-Only, No Mixed Content

upgrade-insecure-requests enforced. All outbound links use rel="noopener noreferrer" to break window.opener leaks.

📦

Minimal, Audited Dependencies

Small, curated dependency tree. Regular audits and pinned versions. No unnecessary runtime code shipped to the client.
